## When to use

Use this page when you are new to blurkit or when you need a fast map of the docs before jumping into a specific runtime, API, or CLI flow.

## Example

```ts title="scripts/encode-hero.ts"
import { encode } from 'blurkit/node'

const result = await encode('./public/hero.jpg', {
  algorithm: 'blurhash',
  size: 32,
})

console.log(result.dataURL)
```

## Inputs / Options / Behavior

- blurkit ships three explicit runtime entrypoints: `blurkit/node`, `blurkit/browser`, and `blurkit/edge`.
- Root import `blurkit` dispatches at runtime and is convenience-first, not bundler-first.
- Successful `encode()` returns `dataURL`, `hash`, `algorithm`, `width`, `height`, and optional `meta`.
- `encodeMany()` is fail-fast and rejects the whole call when one input fails.

## Limits / Caveats

- Node runtime requires `sharp`.
- Browser runtime cannot read local filesystem paths.
- Edge runtime requires `ImageDecoder` and `OffscreenCanvas`.
- Remote URL input depends on network reachability and, in browsers, CORS.
